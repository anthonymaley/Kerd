## Key Decisions
- Modes don't call skills directly. They set session context, present a customizable flow, and track progress. The user invokes each tool. This avoids coupling to GSD's internals.
- Editable checklist before "go": users can skip, reorder, and add custom steps. The mode file is the default, every session can be customized.
- One file per mode in `modes/` directory. Contributors PR a single file. No manifest updates needed.
- Categories group modes: development, business, maintenance, operations.
- Kerd, GSD, and Superpowers are complementary layers: operations, build orchestration, engineering discipline. Modes route between them.
- Hail stays separate from Kerd for now. Strongest future integration point is skriv (writing rules as Hail directives).
